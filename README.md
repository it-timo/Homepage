# dev-timo.com digital ecosystem

A zero-build living archive for Timo Schmidt's software, music, writing, and experiments.

## Source of truth

`WEBSITE_KNOWLEDGE_BASE.md` defines the known identity, work, themes, and website philosophy. Content must not invent dates, releases, project details, links, or biography to make a page appear complete.

## Local development

```bash
python3 tools/build_music_catalog.py
python3 -m http.server 8000
```

Open <http://localhost:8000>.

## Music catalog

The filesystem below `assets/music/` is authoritative. Each album directory contains an `album.json` with album-level metadata and numbered media files:

```text
assets/music/core-override/
├── album.json
├── album_cover.png
├── 01_Used_to_be_easy.wav
├── 01_Used_to_be_easy.png
└── 01_Used_to_be_easy.lrc
```

Run:

```bash
python3 tools/build_music_catalog.py
```

The builder:

- validates every `album.json`
- groups `.wav`, `.png`, and `.lrc` files by track number
- derives titles and URL slugs from filenames
- reads WAV duration through Python's standard library, with `ffprobe` as a fallback
- calculates track count and total album runtime
- generates previous/next track relationships
- writes `data/music_catalog.json`
- creates static `/music/<album>/` and `/music/<album>/<track>/` entry routes
- removes stale routes previously generated by the tool

Adding `14_New_Song.wav`, `14_New_Song.png`, and `14_New_Song.lrc` requires no HTML or JavaScript edit—only a catalog rebuild.

LRC files are loaded on track pages. The browser parser returns metadata, plain lines, and timestamped lines. The current view is intentionally plain; timestamps are retained in `data-time` attributes for future synchronized lyrics.

No autoplay or third-party player is used. Audio playback uses native HTML5 controls for play, pause, and seeking.

## GitHub catalog

GitHub username: **`it-timo`**.

```bash
python3 tools/update_github.py
```

The updater requests public repositories from the GitHub API, excludes forks and archived repositories, fetches languages and README excerpts, sorts by `updated_at` descending, and atomically writes `data/projects.json`.

Set `GITHUB_TOKEN` when needed for higher API limits:

```bash
GITHUB_TOKEN=... python3 tools/update_github.py
```

Feature repositories by adding their exact repository names to `data/project_featured.json`:

```json
{
  "repositories": ["repository-name"]
}
```

Hand-authored engineering context remains separate in `data/project_context.json`; GitHub data is never manually duplicated there.
